《人工智能导论》 大作业
大作业以小组为单位完成。每个小组原则上3-4人， 并由小组协商推选组长，组长负责任
务统筹和确定成员贡献比例（代码仓库的贡献有一定参考） 。
题目： 可解释的谣言检测
1、 目标
基于谣言检测数据集，构建一个检测模型。该模型可以对数据集中的推文进行谣言检测，
同时输出判断依据。 要求：
✓ 输出1： 检测的输出是2分类（0代表非谣言、 1代表谣言） ，分类准确率越高越好；
✓ 输出2： 判断依据的输出是一段文字，解释判断的依据；
✓ 模型具有一定的泛化能力；
✓ 有合理的运行时间。
2、 模型构建要求
✓ 可以使用各种深度学习模型，大语言模型， RAG等技术，构造一个复合的模型。 例
如：用深度学习模型给出检测，大语言模型给出判断依据。
✓ 如果使用大语言模型开发，限定使用我校提供的模型接口，方便复现。我校提供的模
型参见： https://claw.sjtu.edu.cn/guide/sjtu-api/
✓ 模型需要实现的基本功能：输入一个文本，输出一个检测和判断依据。
3、 数据集
从canvas下载rumer2026.zip, 包括一个训练集train.csv和一个测试集val.csv。2
4、 评分标准：满分100分
✓ 大作业报告叙述清楚（30分）
✓ 代码仓库代码可运行，部署说明清楚（25分）
✓ 在val.csv数据集， 检测分类准确率（15分）
✓ 检测依据的可解释性（15分）
✓ 小组分工协作情况（15分）
5、 实现要求
✓ 模型设计及其训练测试过程均需自己实现，可以参考公开的资源，可以参考人工智能
工具的建议和代码；
✓ 组长应该在组队完成的时候，就建立一个公开的代码仓库。后续完成作业过程中，每
个成员都应该积极向代码仓库贡献代码和文档；
✓ 对于雷同作业， 经调查核实后，扣除一定分数；
✓ 实现的模型可以运行， 具有一定的检测性能；
备注： “支持文档” 给出了建议运行环境和深度学习检测参考代码； 判断依据部分没
有给参考代码。
